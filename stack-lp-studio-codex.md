# STACK LP Studio

**作成日時：** 2026-06-12

## 目的

Codexで生成したモバイル向けLP画像を縦に並べ、A/B案を作成し、計測タグ付きの単一HTMLとして出力するプロトタイプ。

## 画面構成

- 左側約1/3：小さく表示したLP全体プレビュー
- 右側約2/3：生成画像ギャラリー、採用順序、画像生成プロンプト
- 右上：HTML書き出し
- 詳細設定：プロジェクト、A/Bテスト、GTMを折りたたみ表示

## LP構造

1. ヒーロー
2. 課題提起
3. 解決策
4. メリット
5. 利用方法
6. 実績・利用者の声
7. CTA

各構造につき2〜3案を生成し、AIおすすめまたはランダムな別案で1画像ずつ選択してLPを構成する。

生成画像は横向きの3:2（1536×1024px）を標準とし、縦長・正方形は使用しない。

## MVP機能

- 複数画像のアップロードとプレビュー
- Codex生成画像を一覧で確認する候補ライブラリ
- 選択中のパートを再生成するボタン
- 画像生成仕様をスクリプト側で3:2・1536×1024pxに固定
- 縦長・正方形画像のアップロード拒否
- ChatGPTログイン済みのCodex App Serverから内蔵画像生成を実行
- 生成画像を`images/generated/`へ保存して候補一覧へ自動追加
- 起動時・画面復帰時・同期ボタンで`images/generated/`を候補一覧へ反映
- 既存LP画像を画風の基準画像として指定
- 色、線、人物表現、文字組みなどの画風ルールをUIから編集
- プロジェクト状態を`projects/`へJSON保存
- 採用画像をBase64埋め込みした単一HTMLを`exports/`へ保存
- 候補画像の拡大表示と選択追加
- ドラッグ、上下ボタンによる並べ替え
- セクションの複製と削除
- Variant A / B の個別編集と複製
- A/B配信比率の設定UI
- GTM IDとCTAイベント名の設定
- CTAリンク付きLP HTMLの書き出し
- LocalStorageへの一時保存

## 起動方法

Codexでこのフォルダを開き、ターミナルから次を実行する。

```bash
node server.mjs
```

その後、Codex内ブラウザで次を開く。

`http://127.0.0.1:8765/stack-lp-studio-codex.html`

ChatGPTでCodexへログイン済みであれば、別途OpenAI APIキーを設定せずに画像再生成を利用できる。

## プロジェクト別フォルダ

ペルソナごとに新規プロジェクトを作り、次の構造で保存する。

```text
projects/
└── <プロジェクトID>/
    ├── project.json
    ├── images/
    │   └── generated/
    └── exports/
```

- `project.json`：候補画像、採用順、A/B案、ペルソナ、画風設定
- `images/generated/`：そのプロジェクトで生成した画像
- `exports/`：画像を内包した完成HTML
- `images/stack-lp/`：全プロジェクト共通の画風基準画像

プロジェクトIDは初回保存時のLP名から作成し、その後LP名を変更しても同じフォルダを使う。

## ペルソナ別の運用

1. 「新規プロジェクトを作成」を押す
2. LP名とペルソナ名を入力する
3. 基準画像と画風ルールを設定する
4. セクション画像を生成・選定する
5. 作業途中は「プロジェクト保存」を押す
6. 完成時に「HTMLを書き出す」を押す

LP名は`商品・施策名 - ペルソナ略称`の形式を推奨する。

例：

- `STACK講座 - 副業30代`
- `STACK講座 - フリーランス初級`
- `STACK講座 - 法人研修担当`

同じペルソナで大幅に訴求を変える場合は別プロジェクトにする。小さなコピーや構成差は同一プロジェクト内のVariant A/Bで管理する。

旧形式の`projects/*.json`、`images/generated/`、`exports/`は移行確認用として残すが、今後の新規保存には使用しない。

## 画像再生成フロー

1. 再生成するLP構造を選択
2. 「選択中のパートを再生成」を押す
3. Codex App Serverがサブスク利用枠で内蔵画像生成を実行
4. 生成画像が候補一覧へ自動追加される
